# 内容模板体系

> 规划目录和写章节时读取此文件。

---

## 全书三层架构

所有实操类知识产品遵循相同的三层递进结构：

```
第一层：认知建立（约 30-40% 篇幅）
├─ 为什么这个领域正在变化（趋势/危机/机遇）
├─ 真实图景（能力边界、常见误区）
└─ 你能/不能做什么（风险评估）

第二层：工具方法（约 20-30% 篇幅）
├─ 4-8 个核心工具/方法（每个 1-2 节）
├─ 每个工具：定义 → 场景 → 步骤 → 案例 → 小结
└─ "拿来就用"的实操性

第三层：场景库（约 30-40% 篇幅）
├─ 20-100 个具体场景（严格模板化）
├─ 每个场景：痛点 → 解法 → 步骤 → 效果数据
└─ 读者按需选 2-3 个快速上手
```

**完整骨架**：
```
前言 → 认知篇（2-3 章）→ 工具篇（2-4 章）→ 场景库（1-2 章）→ 行动篇（1 章）→ 结语 → 附录
```

---

## 前言模板（四层递进，1500-2500 字）

在 1500-2500 字内完成「建立信任 → 说透价值 → 引导阅读」。

### 第一层：破冰故事（300-500 字）

```markdown
# 前言

[具体人物名]，[年龄/身份]，[具体背景 2-3 句]。

[困境描述 2-3 句，用细节让读者代入]。

[转折点：某个事件/发现/行动]。

[结果：量化的改变]。
```

> **去 AI 味要点**：开场人物必须有名字、有年龄或身份、有具体数字。"某位企业家"不合格，"老武，35 岁，在互联网公司干了 8 年"合格。

### 第二层：三句话概括价值（150-300 字）

```markdown
## 这本书是什么

**这本书是[一句话定位]。**

这本书**不是**[排除歧义 1]，也**不是**[排除歧义 2]。

你可以把它当作[使用方式]，[具体使用场景]。
```

> **去 AI 味要点**："不是什么"比"是什么"更重要——它砍掉读者的错误预期。

### 第三层：内容梗概（5-8 条 bullet list）

```markdown
## 你会在这本书中找到

- **第 1-3 章**：[认知篇概述]
- **第 4 章**：[场景库/赛道库——通常是最大卖点]
- **第 5-6 章**：[工具/方法篇]
- **附录**：[速查表/模板/清单]
```

### 第四层：AI 辅助声明（约 100 字，固定文案）

```markdown
> **AI 辅助声明**：本书在资料整理、数据分析和初稿生成过程中使用了 AI 辅助工具。
> 所有内容均经过团队审核、事实核查和独立判断。最终文责由作者承担。
```

## 认知篇章模板（单章 3500-5000 字）

```markdown
# 第 X 章 [章标题]

[题引：1-2 句话概括本章核心问题]

## X.1 [从一个故事/场景切入]

[案例故事：具体人物 + 具体困境 + 转折，300-800 字]

**[金句：提炼故事中的核心洞察]**

## X.2 [核心概念或框架]

[概念定义——零术语版，200-300 字]

| 维度 | 旧认知 | 新认知 |
|------|--------|--------|
| ... | ... | ... |

## X.3 [深化：能/不能、风险/机遇]

- **好消息**：[机遇面]
- **坏消息**：[风险面]
- **关键变量**：[决定成败的因素]

## 本章小结

[3-5 条 bullet point 或 1 个总结表格]

*下一章我们来看[下章核心问题]。*
```

> **必备要素**：故事/案例 + 对比表 + "好消息/坏消息" + 本章小结 + 章尾过渡

## 工具篇章模板（单章 3500-5000 字）

```markdown
# 第 X 章 [工具/方法名称]

[题引]

## X.1 [痛点陈述，200-400 字]

## X.2 [工具定义——零术语版，200-300 字]

## X.3 [3-5 个使用场景，每个 200-400 字]

## X.4 [完整案例：背景→问题→使用过程→量化结果，500-1000 字]

## X.5 [实操清单：3-5 步，可立即执行]

## 本章小结

| 要点 | 说明 |
|------|------|
| ... | ... |

*下一章[承接过渡]。*
```

> **必备要素**：痛点 + 零术语定义 + 场景 + 完整案例 + 实操清单

## 场景库模板

### 单个场景（300-600 字）

```markdown
### [编号] [场景名称]

**痛点**：[2-3 句]
**解法**：[2-3 句]
**切入步骤**：
1. [可立即执行]
2. [第二步]
3. [第三步]

**预期效果**：
- 效率：从 X 小时 → Y 分钟
- 收益：X,XXX-X,XXX 元/月（基于公开数据推算，仅供参考）

**适合人群**：[一句话]
**难度**：★ / ★★ / ★★★
```

### 场景库组织（整章）

```markdown
# 第 X 章 [N 个场景实操]

[使用说明：如何选场景，200-300 字]

## 类别一：[类别名]（场景 1-N）
[类别概述 100-200 字]
### 场景 1 ...
### 场景 2 ...

## 类别二：[类别名]（场景 N+1-M）
...
```

## 结语模板（500-1000 字）

```markdown
# 结语

[回扣前言故事：2-3 句呼应]

[时代意义：为什么此刻行动，2-3 段]

[行动号召：具体的第一步，1-2 段]

**[全书总金句]**
```

> **去 AI 味要点**：结语最后一句必须是具体行动，不是抽象总结。

## 写作要素规范

### 金句

**频率**：每 800-1000 字 1 条，每章 2-4 条。位置：章节开头或论述转折处。

**四种生成公式**：

| 类型 | 公式 | 示例 |
|------|------|------|
| 对比型 | 从 X 到 Y | "从'卖时间'到'卖方法论'" |
| 等式型 | A + B = C | "1 个人 + AI 工具 = 一家公司的战斗力" |
| 反转型 | 不是 X，而是 Y | "不是'会不会被替代'，而是'怎么用 AI 升级'" |
| 强调型 | 关键词 + 修饰 | "你的价值在于专业判断和质量把控" |

**Markdown 写法**（独立段落，不含冒号，触发语义后处理 H1 规则自动标记）：
```markdown
**AI不是替代人，而是让每个人都能拥有专家级的协作伙伴。**
```

### 数据标注（铁律）

所有量化数据必须带免责标注：
```markdown
月收入预期：5,000-10,000 元（基于公开数据推算，仅供参考）
```

常用标注语：
- `（基于公开数据推算，仅供参考）`
- `（数据时效至 20XX 年 X 月）`
- `（具体因行业、地区、个人能力差异）`

### 章间衔接

**悬念过渡**（正文末尾）：
```markdown
但你可能会问：**具体在哪些场景中落地**？
```

**过渡语标记**（触发语义后处理 H3 规则）：
```markdown
*下一章我们来看具体的 30 个实操场景。*
```

## 内容完整性检查清单

- [ ] 前言：破冰故事 + 价值概括 + 内容梗概 + AI 声明
- [ ] 认知篇每章：故事/案例 + 对比表 + 好消息坏消息 + 小结
- [ ] 工具篇每章：痛点 + 零术语定义 + 场景 + 完整案例 + 实操清单
- [ ] 场景库每条：痛点 + 解法 + 步骤 + 效果数据 + 难度评级
- [ ] 所有量化数据标注免责声明
- [ ] 金句频率：每 800-1000 字 1 条，每章 2-4 条
- [ ] 章尾有过渡语衔接下一章
- [ ] 结语回扣开篇 + 行动号召
- [ ] 附录含速查表/操作清单/术语表
